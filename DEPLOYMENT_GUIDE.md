# Deployment Guide for AkkiMate

This app is ready to be deployed to **Vercel** (recommended for Vite apps).

## Prerequisites

- A GitHub account (already set up: `https://github.com/Ro-shni/akkhimate`)
- A Vercel account (free at https://vercel.com)
- Your Google API key for the Gemini AI service

## Deployment Steps

### Step 1: Install Vercel CLI (Optional but Recommended)
```bash
npm install -g vercel
```

### Step 2: Deploy to Vercel

#### Option A: Using Vercel CLI (Recommended)
```bash
npm run deploy
# Or manually: 
# vercel --prod
```

The first time you run this:
- Vercel will ask you to authenticate with your Vercel account
- It will link your project automatically
- It will deploy your project

#### Option B: Using Vercel Web Dashboard
1. Go to https://vercel.com/dashboard
2. Click "Add New Project"
3. Select "Import Git Repository"
4. Select your GitHub repository: `Ro-shni/akkhimate`
5. Click "Import"
6. Set up environment variables (see Step 3)
7. Click "Deploy"

### Step 3: Configure Environment Variables

After creating the project on Vercel, you need to add your API key:

1. Go to your project on Vercel dashboard
2. Settings → Environment Variables
3. Add the following variable:
   - **Name**: `VITE_GOOGLE_API_KEY`
   - **Value**: Your Google API key (from Google AI Studio)
   - **Environments**: Select Production, Preview, and Development

### Step 4: Verify Deployment

Once deployed:
- Your app will be available at `https://akkhimate.vercel.app`
- Vercel will automatically deploy new changes when you push to the `main` branch
- You can check deployment status in the Vercel dashboard

## Troubleshooting

### "VITE_GOOGLE_API_KEY is not set"
- Make sure you've added the environment variable in Vercel settings
- Vercel environment variables take effect on new deployments
- Redeploy your project after adding the variable

### Build failures
- Check the Vercel deployment logs for specific errors
- Ensure all dependencies are properly installed: `npm install`
- Verify the build works locally: `npm run build`

### CSS Import Warnings
- These have been fixed in the latest commit
- Google Fonts are now imported directly in `index.html`
- Tailwind CSS imports are properly ordered in `src/index.css`

## Available Commands

```bash
npm start      # Start development server on port 3000
npm run dev    # Same as npm start
npm run build  # Build for production
npm run deploy # Build and deploy to Vercel (requires Vercel CLI)
npm run lint   # Check TypeScript types
```

## Project Structure

- `src/` - React components and TypeScript source
- `dist/` - Production build output (generated by `npm run build`)
- `.env` - Local environment variables (Git ignored)
- `.env.example` - Template for environment variables
- `vercel.json` - Vercel deployment configuration

## Next Steps

1. Make sure your `.env` file has your API key for local development
2. Run `npm run build` to verify the build works
3. Run `npm run deploy` or use Vercel web dashboard to deploy
4. Test your app at the Vercel URL
5. Push future changes to `main` branch for automatic deployments
